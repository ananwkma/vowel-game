/* ============================================================
   SECTION: MOBILE TOUCH SUPPRESSION

   Suppresses iOS long-press loupe, tap highlight, and zoom.

html, body {
  -webkit-touch-callout: none;           /* iOS long-press callout menu */

**1. Add responsive block size media queries immediately after the `:root {}` block in the CSS VARIABLES section (after the touch suppression rules added in Task 1):**

**Size math (for reference, do not add as comment):**
- Target mobile screen: 375px
- Mobile padding: 12px left + 12px right = 24px total
- Available width: 375 - 24 = 351px
- 7 blocks + 6 gaps: (7 × 42) + (6 × 9) = 294 + 54 = 348px ≤ 351px ✓
- Desktop default stays at 52px (current value in :root)

Add the following media queries:
```css
/* Responsive block sizing — shrinks on mobile, full size on desktop.
   Picker scales proportionally via the same --block-size variable.
   Mobile breakpoint: max-width 425px covers iPhone 6-15 and SE. */
@media (max-width: 425px) {
  :root {
    --block-size: 42px;  /* 7 blocks × 42px + 6 gaps × 9px = 348px; fits 351px available (375px - 24px padding) */
    --block-gap: 7px;    /* Slightly tighter gap on mobile for breathing room */
  }

  #app {
    padding: 24px 12px;  /* Reduce horizontal padding on mobile to maximize block space */
  }
}

@media (max-width: 360px) {
  :root {
    --block-size: 38px;  /* Smallest phones (Galaxy A, older iPhones) */
    --block-gap: 6px;
  }

  #app {
    padding: 20px 8px;
  }
}
```

  <action>
In SECTION 2: WORD ENGINE (around line 608), `WordEngine.wordList` is currently assigned directly to the `WORDS` constant. The WORDS array contains words up to 8+ letters (ABSOLUTE, ACADEMIC, etc.) which would exceed the 7-block single-line layout on mobile.

Per CONTEXT.md decision: "Filter the word list to a maximum of 7 letters (may reduce to 6 after user testing if 7 feels tight)."

**Change the `wordList` property in the `WordEngine` object:**

Find:
```javascript
const WordEngine = {
  // Reference the pre-curated word list
  wordList: WORDS,
```

Replace with:
```javascript
const WordEngine = {
  // Filter to max 7 letters — ensures 7 blocks fit on a single line on mobile (375px screen).
  // User may reduce to 6 after mobile testing if 7 feels tight (see 05-CONTEXT.md).
  wordList: WORDS.filter(w => w.length <= 7),
```

Also update the `isValidGameWord` validation method's length check to match. Find:
```javascript
      isValidGameWord(word) {
        const upper = word.toUpperCase();
        return upper.length >= 4 && upper.length <= 10 && /[AEIOU]/.test(upper);
      },
```

Replace with:
```javascript
      isValidGameWord(word) {
        const upper = word.toUpperCase();
        return upper.length >= 4 && upper.length <= 7 && /[AEIOU]/.test(upper);
      },
```

**No other changes needed.** The `WordSet` (used for win validation) is built from `WORDS` (the unfiltered array) — this is CORRECT and intentional. The player can still construct any valid word using the consonants from a filtered puzzle word, including longer words if consonants happen to permit it. The filter only controls which puzzle words are selected, not which answers are accepted.
