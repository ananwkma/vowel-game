Replace this block (and add urlParams declaration before it) with:

```javascript
const urlParams = new URLSearchParams(window.location.search);
const _overrideDate = urlParams.get('date'); // e.g. ?date=2026-03-01

const DATE_SEED = (() => {
  const targetDate = _overrideDate ? new Date(_overrideDate) : new Date();
  const year = targetDate.getUTCFullYear();
  const month = String(targetDate.getUTCMonth() + 1).padStart(2, '0');
  const day = String(targetDate.getUTCDate()).padStart(2, '0');
  return `${year}-${month}-${day}_ladder_v1`;
})();
```

Explanation of the mapping:
- `path.length` = number of words in the path (including start and end)
- `path.length 4` = 3 steps (e.g., SCARE → STARE → STALE → STILL = 3 changes, 4 words)
- `path.length 5` = 4 steps (e.g., 4 letter changes, 5 words)
- Old constraint: path.length 5-7 = 4-6 steps (too long)
- New constraint: path.length 4-5 = 3-4 steps (cognitively tractable)

The common-word path constraint (`commonPath.length > 11`) stays the same — it ensures the puzzle is solvable via familiar words within 10 steps regardless of optimal path length.

The fallback `{ start: 'SCARE', target: 'STILL', ... }` stays unchanged. The SCARE→STARE→STALE→STALL→STILL path is 4 steps (path.length 5), which fits within the new constraint.
