<objective>
Integrate the drag-and-drop functionality with the Vowel Picker UI. On a successful drop, the picker will appear. When a vowel is selected, the block's content will update, and the picker will disappear.

Purpose: This plan connects the two previously independent features into a single, cohesive user interaction, fulfilling the final part of the core gameplay loop (CORE-06).
Output: An updated `game.html` with the complete, interactive block manipulation and vowel selection flow.
</objective>

<task type="auto">
  <name>Task 1: Trigger Vowel Picker on Drop</name>
  <files>game.html</files>
  <action>
    1.  **Modify the `pointerup` handler:** In the script handling drag-and-drop, locate the `pointerup` event listener.
    2.  **Call `showVowelPicker`:** At the point where a successful drop is confirmed, instead of just releasing the block, call the `showVowelPicker(droppedElement)` function created in the previous plan.
    3.  **Position the Picker:** Modify `showVowelPicker` to accept the dropped element as an argument. Inside the function, calculate the `top` and `left` coordinates based on the dropped element's position (`getBoundingClientRect()`) and update the picker's style to position it adjacent to the block.
  </action>
  <verify>
    - Open `game.html` in a browser.
    - Drag and drop a yellow block.
    - The vowel picker should immediately appear next to the block you just dropped.
  </verify>
  <done>
    The Vowel Picker is now displayed in the correct position upon a successful block drop.
  </done>
</task>

<task type="auto">
  <name>Task 2: Update Block Content on Vowel Selection</name>
  <files>game.html</files>
  <action>
    1.  **Refactor the Vowel Picker:** Modify the vowel picker's selection logic. Instead of just logging to the console, it needs to communicate the selected vowel back to the block that triggered it.
    2.  **Use a Callback or Promise:** A good pattern is to have `showVowelPicker` return a Promise that resolves with the selected vowel's value. The `pointerup` handler can then `await` this promise.
        -   Modify `showVowelPicker` to store `resolve` and `reject` functions from a new `Promise`.
        -   When a vowel is selected (click or Enter), call the stored `resolve` function with the vowel's value.
        -   If the picker is closed with `Escape`, call the `reject` function.
    3.  **Update `pointerup` Handler:**
        -   Wrap the call to `showVowelPicker` in a `try...catch` block.
        -   Use `const selectedVowel = await showVowelPicker(droppedElement);`.
        -   On resolution, set `droppedElement.textContent = selectedVowel;`.
        -   The `catch` block will handle the Escape key being pressed, so it can be left empty.
  </action>
  <verify>
    - Drag and drop a yellow block. The picker appears.
    - Select 'E' from the picker using either a click or keyboard.
    - The picker should disappear, and the yellow block's text should now be 'E'.
    - Repeat for another block and a different vowel.
  </verify>
  <done>
    Selecting a vowel updates the corresponding block's display, completing the interaction loop.
  </done>
</task>
