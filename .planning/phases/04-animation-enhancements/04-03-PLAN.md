<task type="auto">
  <name>Add CSS for Block Swipe-in Animation</name>
  <files>game.html</files>
  <action>
    Locate the `<style>` block in `game.html`.
    Add a new `@keyframes` rule named `swipeIn` that moves elements from an off-screen position (e.g., `translateX(-100%)` or `opacity: 0` combined with `translateX(-20px)`) to their final `transform: translateX(0)`.
    Define a new CSS class, for example, `.block-swipe-in`, that applies this `swipeIn` animation with a default duration (e.g., 0.4s) and `forwards` fill mode.
    Example keyframes:
    ```css
    @keyframes swipeIn {
      0% {
        opacity: 0;
        transform: translateX(-50px) scale(0.9);
      }
      100% {
        opacity: 1;
        transform: translateX(0) scale(1);
      }
    }
    .block.block-swipe-in {
      animation: swipeIn 0.4s ease-out forwards;
      opacity: 0; /* Ensure it starts hidden before animation */
    }
    ```
  </action>
  <verify>
    Inspect the `game.html` file to ensure the new `@keyframes swipeIn` and `.block-swipe-in` CSS class are present in the `<style>` block.
  </verify>
  <done>
    The `game.html` file contains CSS to define a swipe-in animation for blocks.
  </done>
</task>
