  <action>
IMPORTANT: The current index.html contains the full VOWEL game. Do NOT delete it yet — Plan 03 will copy it to vowel.html. In this plan, completely overwrite index.html with the hub portal HTML below.

Write index.html as a self-contained hub page that:

1. **Imports** `styles/design-tokens.css` via `<link rel="stylesheet">` as the first stylesheet in `<head>`.

2. **Header section**: `<h1 id="hub-title">Word Games</h1>` and `<p id="hub-tagline">Daily puzzles await</p>` centered above the card grid. The string "Word Games" must appear as a JavaScript-accessible data attribute or CSS custom property placeholder comment so it can be changed without touching code: `data-collection-name="Word Games"` on the `<h1>`.

3. **Card grid**: Three cards in a CSS Grid container (`#hub-grid`) with `grid-template-columns: 1fr` on mobile and `grid-template-columns: 1fr 1fr` at `@media (min-width: 768px)`. Each card:
   - VOWEL card: `<a href="vowel.html" class="game-card" id="card-vowel" data-game="vowel"><h2>VOWEL</h2></a>` — clickable, navigates to vowel.html
   - Word Ladder card: `<div class="game-card disabled" id="card-ladder"><h2>Word Ladder</h2></div>` — NOT an `<a>` tag (not clickable), `disabled` class, `aria-disabled="true"`
   - Letter Hunt card: `<div class="game-card disabled" id="card-hunt"><h2>Letter Hunt</h2></div>` — same treatment

4. **CSS card styles** (inline `<style>` in `<head>`):
   - `.game-card`: padding `var(--spacing-lg)`, background `#FFFFFF`, border-radius `var(--block-radius)`, box-shadow `var(--shadow-md)`, text-align center, text-decoration none, color inherit, cursor pointer
   - Hover on active (non-disabled) cards: Use `filter: brightness(0.95) saturate(1.2)` for amber tint effect. Also add `transform: translateY(-2px)` and enhanced shadow on hover.
   - `.game-card.disabled`: `opacity: 0.45`, `filter: grayscale(80%)`, `cursor: default`, `pointer-events: none`
   - `.game-card.completed`: `opacity: 0.65` (purely visual dimming, no text change — per locked decision)
   - Transitions on all cards: `transition: opacity var(--transition-normal), filter var(--transition-normal), transform var(--transition-normal)`
   - Card `<h2>`: font-size `1.25rem`, color `var(--color-primary-text)`, no margin

5. **Daily completion status script** (inline `<script>` at bottom of `<body>`):
   ```javascript
   (function() {
     try {
       const status = JSON.parse(localStorage.getItem('wordGames_dailyStatus') || '{}');
       const today = new Date().toISOString().split('T')[0];
       ['vowel', 'ladder', 'hunt'].forEach(function(gameId) {
         const record = status[gameId];
         if (record && record.completed && record.dateKey === today) {
           const card = document.getElementById('card-' + gameId);
           if (card) card.classList.add('completed');
         }
       });
     } catch(e) {
       // localStorage unavailable — ignore silently
     }
   })();
   ```

6. **Page transition**: Add a CSS `@keyframes fadeIn` and apply `animation: fadeIn 0.3s ease-out` to `body` so the hub fades in when navigated back from a game. This is a simple CSS approach — no JS, no View Transitions API (broad compatibility).

7. **Meta tags**: Include standard `<meta charset="UTF-8">`, `<meta name="viewport" content="width=device-width, initial-scale=1">`, and `<title>Word Games</title>`.

8. **Max-width container**: Wrap header + grid in `<div id="hub-container">` with `max-width: 600px; margin: 0 auto;` so the hub doesn't sprawl on ultra-wide screens.

The hub must NOT contain any VOWEL game code, word lists, or game logic — purely hub navigation UI.

localStorage namespace comment at top of script:
```javascript
// localStorage namespace:
// - wordGames_dailyStatus: { vowel: { completed, dateKey, timestamp }, ... } — shared hub key
// - vowel_*, ladder_*, hunt_* prefixes: game-specific state (owned by each game)
```
