What was built:
- Express server (server/index.js) serving the game from public/ and the API on /api/
- SQLite backend (server/db/scores.db) with 10 seeded test scores for today's date (from Plan 01)
- score submission via ApiService.submitScore() — POST /api/scores on puzzle completion
- Real-time percentile display via ApiService.fetchStats() — GET /api/stats after completion
- Graceful fallback to PerformanceStats.getPercentile() simulation when backend is unreachable
- ?debug URL param to bypass already-played guard for re-testing
  </action>
  <how-to-verify>
**Pre-check:** Server must be running. Confirm terminal shows "Server running on http://localhost:3000".

**Step 1: Open the game**
Navigate to http://localhost:3000/?debug in your browser (use ?debug to allow re-play if you already played today).
- Expected: Game loads with 5 letter blocks visible, "VOWEL" header, timer at 0:00.

**Step 2: Play through a word**
- Drag-drop or tap to arrange blocks and select a vowel.
- Expected: Dragging works, vowel picker appears, selecting a vowel triggers win/lose state with board color change.

**Step 3: Complete all 5 words**
- Play through all 5 daily words (use Give Up on some to speed up).
- Expected: Timer counts up including penalty seconds for Give Ups; pip dots fill in green (solved) or salmon (gave up); "Word X of 5" counter advances.

**Step 4: Observe the results screen**
- Expected: A results overlay appears immediately after word 5 with "Loading stats..." text.
- Within 1-2 seconds, the rank text updates to something like "Better than 45% of players today (11 today)".
- The totalPlayers count should be >= 11 (10 seeded + your own submission).
- The percentile must be a real number between 1-99 (not the fallback simulation value).

**Step 5: Check server logs**
- Switch to the terminal running the server.
- Expected: Two log lines — one for score submission (`POST /api/scores 201`) and one for stats (`GET /api/stats 200`).

**Step 6: Reload to confirm no double-submission**
- Refresh the browser (F5).
- Expected: Results screen re-displays immediately (already-complete guard fires).
- Expected: Server logs show only a GET /api/stats request (no second POST /api/scores).

**Step 7: Test offline fallback**
- Open DevTools → Network tab → check "Offline" or right-click requests and block localhost:3000.
- Reload using http://localhost:3000/?debug.
- Complete the puzzle again.
- Expected: Rank text shows a simulated percentile ("Better than ~50% of players") or "Stats unavailable" — not a blank or crash.

**Regression checks (v1.0/v1.1 features):**
- Timer counts up from 0:00 and persists across page reloads mid-puzzle.
- Give Up button shows countdown from 60; pressing it adds penalty seconds to timer.
- Give Up at 0 is free (no penalty).
- 5 pip dots in header fill correctly: green for solved, salmon for gave up.
- Blocks animate on new word load (swipe-in stagger) and bounce on drop (settle animation).
  </how-to-verify>
