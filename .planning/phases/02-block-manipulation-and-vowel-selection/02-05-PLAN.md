<task type="auto">
  <name>Task 1: Restrict Drag Movement and Implement Dynamic Spacing</name>
  <files>game.html</files>
  <action>
    1.  **Restrict Vertical Movement:** Modify the `pointermove` handler for drag-and-drop. Only update the `translateX` value of the dragged element; ensure `translateY` remains constant (e.g., `0` or `initial` relative to its starting Y position).
    2.  **Implement Dynamic Spacing (Insertion Logic):**
        -   In the `pointermove` handler, as the `draggedElement` moves horizontally, iterate through all other blocks (`.block` elements on the board).
        -   Determine if the `draggedElement` is currently "over" a valid insertion point (i.e., between two existing blocks, or at the start/end).
        -   Visually indicate potential drop zones (e.g., by temporarily adjusting the margin or adding a placeholder element).
        -   When `pointerup` occurs, instead of simply appending or inserting, use `insertBefore` or `appendChild` to place the `draggedElement` into the correct position in the DOM, creating a gap for it.
        -   Ensure blocks shift correctly to accommodate the inserted vowel. This might involve setting `display: flex; gap: var(--block-gap);` on the `#game-board` and letting CSS handle spacing, or more complex JS for direct element manipulation. Prioritize letting flexbox handle it if possible.
    3.  **Limit Horizontal Movement:** Ensure that the drag operation only allows dropping within the confines of the main `#game-board` element, or at least logically within the "word string" area defined by the consonant blocks.
  </action>
  <verify>
    - Open `game.html` in a browser.
    - Drag a yellow block: it should only move horizontally.
    - Drag a yellow block between consonants: the consonants should visibly shift to make space.
    - Drop a yellow block: it should insert itself into the correct position.
  </verify>
  <done>
    Drag movement is restricted to horizontal, and blocks dynamically make space for drops.
  </done>
</task>

<task type="auto">
  <name>Task 2: Implement Right-Click/Long-Press for Vowel Picker Activation</name>
  <files>game.html</files>
  <action>
    1.  **Modify Vowel Picker Activation:** Change the `pointerup` handler logic. Instead of automatically showing the vowel picker on *any* drop, it should *only* show when a specific action is performed on the *vowel block itself*.
    2.  **Right-Click (Context Menu) for Desktop:**
        -   Add a `contextmenu` event listener to the vowel blocks.
        -   Prevent the default browser context menu.
        -   Call `vowelPicker.show(targetElement)` using the block that was right-clicked.
    3.  **Long-Press for Touch:**
        -   Implement a long-press detection mechanism for touch devices on vowel blocks. This typically involves:
            -   `touchstart`: record timestamp.
            -   `touchend`: clear timer.
            -   `touchmove`: if significant movement, cancel long-press.
            -   If `touchstart` duration exceeds a threshold (e.g., 500ms) without `touchmove`, trigger `vowelPicker.show(targetElement)`.
        -   Ensure this doesn't conflict with normal drag operations. A `pointerdown` event listener (which covers both mouse and touch) should be modified to differentiate between a drag start and a long-press that triggers the picker. If `e.button === 2` (right-click) or a long-press is detected, show the picker instead of initiating a drag.
    4.  **Remove Automatic Picker on Drop:** Remove any existing logic that automatically displays the picker on a `pointerup` after a drag.
  </action>
  <verify>
    - Open `game.html` in a browser.
    - Right-click a vowel block: the picker should appear.
    - Perform a long-press on a vowel block on a touch-enabled device/emulator: the picker should appear.
    - Normal drag-and-drop should *not* automatically show the picker anymore.
  </verify>
  <done>
    Vowel picker activation is now triggered by right-click/long-press on a vowel block.
  </done>
</task>
