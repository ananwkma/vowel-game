<task type="checkpoint:human-verify" gate="blocking">
  <name>Verify Block Manipulation and Vowel Selection</name>
  <what-built>
A complete interactive loop for placing vowel blocks and selecting their letters. This includes:
1.  Drag-and-drop for yellow blocks using Pointer Events.
2.  An accessible Vowel Picker UI that appears on drop.
3.  Integration logic that updates the block's content upon vowel selection.
  </what-built>
  <how-to-verify>
Please open the `game.html` file in your browser and test the following scenarios:

**Scenario 1: Mouse/Touch Interaction**
1.  Click and hold a yellow block. It should lift up visually (e.g., with a shadow).
2.  Drag it horizontally between two consonant blocks and release.
3.  **Expected:** The block should settle in the new position, and the A/E/I/O/U vowel picker should appear next to it.
4.  Click the vowel 'I' in the picker.
5.  **Expected:** The picker should disappear, and the block should now display the letter 'I'.
6.  Repeat this process for the other yellow block, choosing a different vowel.

**Scenario 2: Keyboard Accessibility**
The research for this phase included keyboard-only interaction patterns. While not explicitly built in Plans 1-3, we need to verify the state of accessibility.
1.  Use the `Tab` key to try to navigate to the yellow blocks.
2.  If you can focus a block, press `Enter` or `Space` to try and "pick it up".
3.  Press `Tab` again to try and move to a drop location.
4.  Press `Enter` to "drop" it.
5.  **Question:** Is any part of this keyboard flow working? If not, that is acceptable for now, but we need to record it. The core logic is pointer-based. A future plan can add the keyboard state machine from the research.

**Scenario 3: Edge Cases**
1.  Drag a block and press the `Escape` key while dragging (if possible in your browser). Does it cancel gracefully?
2.  Drag a block and drop it back into its original position. Does the vowel picker still appear?
3.  Rapidly drag and drop blocks. Does the UI keep up without errors?
  </how-to-verify>
  <resume-signal>
Type "approved" if the functionality is correct and feels good. If there are issues, please describe them (e.g., "The picker appears in the wrong place," "Selecting a vowel doesn't work on the second block").
  </resume-signal>
