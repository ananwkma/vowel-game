<task type="auto">
  <name>Task 1: Show/Hide Picker During Drag and Update Position</name>
  <files>game.html</files>
  <action>
    1.  **Modify `vowelPicker` module:**
        -   Add a `moveWithTarget(targetElement)` method to the `vowelPicker` module. This method should take a `targetElement` (the currently dragged block) and continuously update the picker's `top` and `left` CSS properties to center itself over that `targetElement`.
        -   Modify the existing `show()` function to also call `moveWithTarget()` initially.
    2.  **Integrate with Drag Logic (`pointerdown`, `pointermove`, `pointerup`):**
        -   **`pointerdown`:** Once a draggable vowel block is identified and a drag operation is initiated (i.e., it's not a right-click/long-press), call `vowelPicker.show(currentDraggable)` to make the picker appear.
        -   **`pointermove`:** While dragging (`currentDraggable` is not null), continuously call `vowelPicker.moveWithTarget(currentDraggable)` to ensure the picker follows the dragged block.
        -   **`pointerup`:** Immediately after the `currentDraggable` is released (and the DOM reordering is done), call `vowelPicker.hide()` to make the picker disappear.
    3.  **Remove previous activation logic:** Remove any logic that was previously showing the picker on right-click/long-press (from `contextmenu` listener and long-press timer). This functionality is now replaced by the drag-initiated show.
  </action>
  <verify>
    - Open `game.html` in a browser.
    - Click and hold a yellow block and start dragging. The vowel picker should appear immediately over the block.
    - As you drag, the picker should follow the block, staying centered.
    - Release the mouse button. The picker should immediately disappear.
    - Right-click a vowel block. The picker should NOT appear.
  </verify>
  <done>
    The vowel picker now correctly appears during drag, follows the block, and disappears on drop.
  </done>
</task>

<task type="auto">
  <name>Task 2: Select Vowel During Drag and Update Block</name>
  <files>game.html</files>
  <action>
    1.  **Modify `vowelPicker.selectVowel()`:**
        -   The `selectVowel` function in the `vowelPicker` module currently resolves a promise and hides the picker. This needs to happen *while* the block is still being dragged.
        -   The `vowelPicker` module needs to somehow communicate the selected vowel back to the currently dragged block. Instead of a promise, `vowelPicker` could expose a `setSelectedVowelCallback(callback)` method.
        -   When `selectVowel` is called, it should invoke this callback with the selected vowel.
    2.  **Integrate with Drag Logic:**
        -   In the `pointerdown` handler (where drag is initiated), before calling `vowelPicker.show()`, set up the callback: `vowelPicker.setSelectedVowelCallback((vowel) => { currentDraggable.textContent = vowel; });`.
        -   This will ensure that when a vowel is selected from the picker *during* drag, the `currentDraggable`'s `textContent` is updated immediately.
  </action>
  <verify>
    - Open `game.html` in a browser.
    - Drag a yellow block. The picker appears.
    - Click a vowel in the picker. The dragged block's letter should change *immediately* while you are still dragging it.
    - Release the mouse. The block drops with the newly selected letter.
  </verify>
  <done>
    Vowel selection now updates the dragged block's letter in real-time.
  </done>
</task>
