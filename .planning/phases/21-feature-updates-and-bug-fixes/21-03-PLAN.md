  <action>
Update ladder.html to use rolling-window seeding. The current DATE_SEED (lines ~941-947) uses `${year}-${month}-${day}_ladder_v1`, and getDailyLadderPuzzle (lines ~949-986) uses a 50-attempt loop where each attempt seeds with `DATE_SEED + '_' + attempt`.

The fix is minimal: update DATE_SEED to include a 150-day window offset, so that consecutive days produce meaningfully different base seeds, breaking the hash collision pattern.

**Step 1 — Add daysSinceEpoch helper** immediately above the `const DATE_SEED` IIFE (around line 941):

```javascript
function daysSinceEpoch(date) {
    const epoch = new Date('2000-01-01T00:00:00Z');
    return Math.floor((date - epoch) / (24 * 60 * 60 * 1000));
}
```

**Step 2 — Update DATE_SEED** (replace the existing IIFE, keeping all UTC methods):

Old:
```javascript
const DATE_SEED = (() => {
    const targetDate = _overrideDate ? new Date(_overrideDate) : new Date();
    const year = targetDate.getUTCFullYear();
    const month = String(targetDate.getUTCMonth() + 1).padStart(2, '0');
    const day = String(targetDate.getUTCDate()).padStart(2, '0');
    return `${year}-${month}-${day}_ladder_v1`;
})();
```

New:
```javascript
const DATE_SEED = (() => {
    const targetDate = _overrideDate ? new Date(_overrideDate) : new Date();
    const year = targetDate.getUTCFullYear();
    const month = String(targetDate.getUTCMonth() + 1).padStart(2, '0');
    const day = String(targetDate.getUTCDate()).padStart(2, '0');
    const days = daysSinceEpoch(targetDate);
    const windowOffset = Math.floor(days / 150); // Rotates every 150 days
    return `${year}-${month}-${day}_ladder_v2_offset_${windowOffset}`;
})();
```

That is the only change required. The getDailyLadderPuzzle() function seeds each attempt as `DATE_SEED + '_' + attempt` (e.g., `_0`, `_1`, ... `_49`). With the new DATE_SEED that includes the day and window offset, each day gets a unique base seed, which means attempt 0 on 2026-03-01 and attempt 0 on 2026-03-02 now use different seeds. The 50-attempt fallback loop remains unchanged.

Do NOT change getDailyLadderPuzzle() or any other logic. The seeding collision fix is entirely in DATE_SEED.

**Why this works for Ladder but not Cipher/Hunt:** Ladder's attempt-loop pattern already provides variety within a day (tries 50 different seed variants). The problem was that the base part of the seed (DATE_SEED) was too similar for consecutive days. Adding the window offset creates enough distance between consecutive day seeds to avoid collision. Cipher and Hunt needed Fisher-Yates because they had a single index pick with tiny corpora; Ladder's multi-attempt loop is more robust.

**Version bump note:** The seed suffix changes from `_ladder_v1` to `_ladder_v2_offset_N`. Players see a different puzzle on their next visit (one-time, intentional).
