Implement the core drag-and-drop mechanics for the yellow vowel blocks using Pointer Events. This enables players to pick up, move, and place blocks horizontally between the consonant blocks.

Purpose: This is the foundational interaction for the game's core loop, as specified in requirement CORE-03 and CORE-04.
Output: A `game.html` file where yellow blocks are draggable and can be re-ordered.
</objective>

<task type="auto">
  <name>Task 1: Add Drag-and-Drop CSS and Attributes</name>
  <files>game.html</files>
  <action>
    1.  **Modify CSS:** In the `<style>` block of `game.html`, add styles for a `.dragging` class. This class should apply a visual effect like a box-shadow and slightly increase the element's size to give clear feedback that it's being dragged.
    2.  **Add `touch-action`:** To the existing `.block.vowel` CSS rule, add `touch-action: none;`. This is critical to prevent the page from scrolling on touch devices during a drag, as recommended in the research.
    3.  **Add Draggable Attributes:** Ensure the yellow blocks have a `data-draggable` attribute to identify them in the script.
  </action>
  <verify>
    - Inspect the `<style>` block in `game.html` to confirm the presence of the `.dragging` class definition and `touch-action: none;` on `.block.vowel`.
    - Inspect the yellow block elements in the HTML to confirm they have `data-draggable`.
  </verify>
  <done>
    CSS rules and HTML attributes required for the drag interaction are in place.
  </done>
</task>

<task type="auto">
  <name>Task 2: Implement Pointer Event Listeners</name>
  <files>game.html</files>
  <action>
    In a `<script>` tag at the bottom of `game.html`:
    1.  **Add `pointerdown` listener:**
        - Attach a `pointerdown` event listener to the vowel blocks.
        - Inside the handler, check for the main button (`e.button !== 0`).
        - Set a state variable, e.g., `let draggedElement = null;`.
        - On `pointerdown`, set `draggedElement` to the target element.
        - Add the `.dragging` class to the element.
        - CRITICAL: Call `draggedElement.setPointerCapture(e.pointerId)` to ensure consistent event flow.
    2.  **Add `pointermove` listener:**
        - Attach a `pointermove` listener to the `document`.
        - If `draggedElement` is set, calculate the new horizontal position.
        - Update the element's position using `draggedElement.style.transform = 'translateX(...)px';`.
        - Implement logic to visually indicate where the block will be dropped (e.g., by shifting other blocks). This can be done by checking the dragged element's position relative to the consonant blocks.
    3.  **Add `pointerup` listener:**
        - Attach a `pointerup` listener to the `document`.
        - If `draggedElement` is set:
        - Determine the correct drop target (the space between consonants).
        - Use `insertBefore` to move the `draggedElement` in the DOM to its new position.
        - Reset the element's transform style.
        - Remove the `.dragging` class.
        - CRITICAL: Call `draggedElement.releasePointerCapture(e.pointerId)`.
        - Reset the state variable: `draggedElement = null;`.
  </action>
  <verify>
    - Open `game.html` in a browser.
    - Press and drag a yellow block. It should move horizontally with the cursor.
    - Release the block between two consonant blocks. The block should move to that position in the row.
  </verify>
  <done>
    Players can successfully drag and drop yellow blocks to re-order them. The interaction is based on Pointer Events as per the research.
  </done>
</task>
