  <name>Task 1: Create seed-test-data.js script</name>
  <files>seed-test-data.js</files>
  <action>
    Create `seed-test-data.js` at the project root. This script POSTs 10 scores to `/api/scores` for today's date using a range of realistic completion times, so that percentile comparisons in the UI are meaningful.

    The script must:
    1. Compute today's date as YYYY-MM-DD using `new Date().toLocaleDateString('en-CA')` (same pattern as the frontend's DailyEngine).
    2. Define an array of 10 test scores in milliseconds spread across a realistic range — e.g. [18000, 25000, 32000, 41000, 55000, 68000, 90000, 120000, 150000, 210000] (18s to 3.5min, covering fast to slow players).
    3. For each score, generate a unique userId like `seed-user-{index}` and POST `{ userId, date, timeMs }` to `http://localhost:3000/api/scores`.
    4. Log each result: `Seeded user-{i}: {timeMs}ms → status {statusCode}`.
    5. After all POSTs, fetch `/api/stats?date={today}&timeMs=55000` and print the returned JSON so the user can confirm percentile logic is working.
    6. Exit with code 1 if any POST returns non-201, or if the stats fetch fails.

    Use `async/await` with a `try/catch` wrapping `runSeed()`. No external dependencies — use Node's built-in `fetch` (available in Node 18+).

    Do NOT delete existing rows before seeding — the script is additive and safe to re-run (duplicate rows are fine for this test purpose, and the schema has no unique constraint on user_id+puzzle_date).
  </action>
  <verify>
    With the server running (`node server/index.js`), run:
    ```

    ```
    Expected: 10 lines of "Seeded seed-user-N: XXXXXms → status 201", followed by stats JSON showing `totalPlayers >= 10` and a `percentile` between 1 and 99.
  </verify>
